## Project Overview

**GeoSítio** — Inteligência Geoespacial Aplicada à Propriedade Rural. A data science and AI portfolio project that applies geotechnology to a real rural property (~3,3 ha) in Monteiro Lobato, SP, Brazil. The project collects, processes, analyzes and visualizes geographic data using satellite imagery, elevation models, climate datasets and vector data.

Final artifact is a static HTML dashboard (`data/sitio.html`, ~14 MB) generated by `gerar_site.py`, with pure HTML + Plotly.js + Leaflet. **No Streamlit** (tested and rejected: not good enough UX).

## Tech Stack

- **Language:** Python 3.10+ (virtualenv in `.venv/`)
- **Geospatial:** rasterio, geopandas, shapely, folium, earthengine-api, leaflet (via folium)
- **Data viz:** Plotly.js, matplotlib (notebooks)
- **ML/AI:** scikit-learn (already used via Earth Engine unsupervised K-Means)
- **Fonts (web):** Geist + Geist Mono (Google Fonts)
- **Image processing:** Pillow + NumPy (for thumbnail cropping)
- **Data sources (active):** Sentinel-2 (ESA), SRTM 30 m (NASA), CHIRPS (chuva, UCSB), ERA5-Land (ECMWF), Esri World Imagery (tiles base)

## Current Version — v3.0

Site has **10 sections** with sticky TOC + scroll-spy and **8 satellite layers** on the map, all with dynamic legends (postMessage bridge between the Folium iframe and the parent site).

**v3.0 (abr/2026)** expandiu a seção Clima com dois blocos novos:
- Série anual de chuva 2015–2024 (CHIRPS soma anual) destacando ano mais seco e mais chuvoso vs média.
- Cruzamento NDVI × chuva mensal (2023–2026) com cálculo de correlação em lag 0/1/2 meses, mostrando que a vegetação responde à chuva com atraso.

### Sections (in order)

1. **Mapa do sítio** — Leaflet with Sentinel-2 RGB, NDVI, Uso do Solo (IA)
2. **Relevo do terreno** — SRTM-derived: EMBRAPA slope classes + aspect (wind rose) + erosion risk (BSI × slope)
3. **Uso do solo** — K-Means classification (5 classes) as a donut + list
4. **Recomendações por zona** — tabs (Pasto verde / Pasto seco / Zona baixa / Mata / Construções) with inline zona-stats
5. **Onde está a água** — TWI (Topographic Wetness Index) + NDWI + coordenadas do ponto ótimo pro tanque + thumbnail estático `data/thumb_agua.png`
6. **Saúde da vegetação** — NDVI série temporal 3 anos (~120 medições)
7. **Ciclo anual** — NDVI sazonal mensal
8. **Clima histórico** — CHIRPS (chuva) + ERA5-Land (temp) 2015–2024 com linha de risco de geada em 5 °C. Inclui série anual de chuva (anos anômalos) e cruzamento NDVI × chuva com análise de lag (v3.0)
9. **Criação animal** — lista editorial padronizada (sem emojis)
10. **Metodologia** — `<details>` com stats brutas + fontes

### Map layers (exclusive, radio-like)

Sentinel-2 · NDVI · Uso do Solo (IA) · TWI · NDWI · Hillshade · EMBRAPA · Risco de erosão

### Design system

- Estilo: **refined minimalism** (inspirado em Vercel / Anthropic / Linear)
- Paleta: zinc-950 + off-white + **lime-600 como único acento vibrante**
- Fontes: **Geist** (UI/display) + **Geist Mono** (números/dados)
- Hero com mesh gradient + grain overlay SVG
- Cards glass (backdrop-blur + border 1px)
- Documento fonte: `design-system/MASTER.md`

### Important decisions / pitfalls

- **URLs de tile do Earth Engine expiram em ~24 h.** O site precisa ser regenerado (`python gerar_site.py`) para renovar os tokens. Documented na seção do mapa em `gerar_site.py`.
- **ERA5-Land (11 km) e CHIRPS (5 km) são MAIORES que o sítio (180 m).** Usar `geometria.centroid(1)` como região no `reduceRegion`, nunca a geometria do sítio (retorna NaN).
- **Paleta de tiles do mapa × legenda do site:** precisam casar EXATAMENTE. A paleta do `gerar_url_tile_uso_solo` em `src/ee_utils.py` é a fonte da verdade — o donut, a tabela e os chips no site devem usar os mesmos hex.
- **Cards de animais NÃO usam emoji** (decisão do Rafael). Lista editorial com nome serif + quantidade em mono + descrição.
- **Thumbnail `data/thumb_agua.png`** é versionado no repo (não regerado automaticamente). Se quiser novo, substituir o arquivo.

## Conventions

- **Código, comentários e texto em português (pt-BR)**, exceto quando referencia API de biblioteca.
- Notebooks prefixados por número: `01_...`, `02_...`, `03_...`.
- **Dados pesados não são commitados.** `.gitignore` cobre: rasters (.tif, .nc), shapefiles grandes, artefato `data/sitio.html` e capturas auxiliares `data/screenshot_*.png`.
- **CRS:** EPSG:4326 (WGS84) para exibição; EPSG:31983 (SIRGAS 2000 / UTM 23S) para análise métrica.
- **Paletas de cor** são controladas por tokens CSS em `:root` no `<style>` do `gerar_site.py`. As paletas do Earth Engine (em `ee_utils.py`) devem casar.

## Quando trabalhar no site

1. Rodar `gerar_site.py` uma vez pra garantir que o pipeline funciona.
2. Pra alterações de CSS/HTML, editar direto o template dentro do `gerar_site.py` e regenerar.
3. Pra mudanças de análise, ajustar `src/ee_utils.py` e regenerar.
4. **Sempre fazer `Ctrl+F5`** no navegador pra evitar cache ao iterar.
